# 视频→博文工作流：使用说明

本仓库把「口播/视频内容」变成「可发布的 Markdown 博文」拆成两段：**本地只做听写**（不占 API），**写作与质检在 Cursor / Codex / Claude Code 里用 SKILL 链完成**。设计理由与完整约束见 **`视频博文工作流-架构版.md`**。

---

## 1. 你在什么场景用哪条入口

| 情况 | 入口 `ENTRY` | 你做什么 |
|---|---|---|
| 手里是 `.mp4` / `.mov` / `.mkv`，需要先出字幕与纯文本 | `video` | 先跑 `video2blog.py`，再在 Agent 里从 Step 3 继续 |
| 已经有逐字稿、纪要、飞书导出文字等 | `transcript` | **不跑**脚本；把稿放进仓库（如 `input/Text/`）或指出路径，Agent 从 Step 3 起跑 |

体裁路由 `ROUTING`（与入口无关）常见：`/lecture`、`/dialogue`、`/screencast`、`/meeting`、`/default`。映射规则见架构版 §三。

---

## 1.5 输入文件根 vs 单次输入文件（必读）

详见 **`memory/CONFIG.md`**。简述：

| 概念 | 用于 | 含义 |
|---|---|---|
| **视频输入文件根** | `ENTRY → video` | 用来解析**相对路径**的基准目录；等价于 **`VIDEO2BLOG_INPUT_ROOT`** 或 **`python video2blog.py --input-root DIR`**。 |
| **单次输入文件** | `ENTRY → video` | **某一个视频文件**的路径（传给脚本的 `VIDEO` 参数）。 |
| **文字稿输入文件根** | `ENTRY → transcript` | 建议把逐字稿集中放在例如 **`input/Text/`**；Agent 的 `SOURCE` 指向具体文件即可（脚本不参与）。 |

## 2. 环境准备（仅 `ENTRY → video` 必装）

1. **ffmpeg**：`brew install ffmpeg`
2. **Python 依赖**：在仓库根目录执行  
   `pip install -r requirements.txt`
3. **mlx-whisper 模型（首选）**：首次转写会从 Hugging Face 拉权重（体量约数 GB），需联网一次。默认模型可由环境变量覆盖：  
   `VIDEO2BLOG_WHISPER_MODEL=<HF 仓库名>`
4. **whisper.cpp（可选 fallback）**：当运行环境拿不到 Metal / MLX 不可用时，可安装：  
   `brew install whisper-cpp`  
   并准备一个 ggml 模型文件，用 `--whisper-cpp-model` 或 `VIDEO2BLOG_WHISPER_CPP_MODEL` 指向它。

**Agent 改写阶段**若只用 Cursor/Codex/Claude Code 内置模型：**不需要**配置 `ANTHROPIC_API_KEY`（除非你自行改脚本去接 API）。

## 3. 本地脚本：听写产出

```bash
# 绝对路径单次文件（与以往相同）
python video2blog.py /path/to/分享.mp4

# 自动引擎：先试 mlx-whisper，失败后默认询问如何处理
python video2blog.py /path/to/分享.mp4 --engine auto

# 授权脚本在普通 macOS Terminal 里跑 mlx-whisper
python video2blog.py /path/to/分享.mp4 --run-in-terminal

# 强制使用 mlx-whisper
python video2blog.py /path/to/分享.mp4 --engine mlx

# 强制使用 whisper.cpp
python video2blog.py /path/to/分享.mp4 \
  --engine whisper-cpp \
  --whisper-cpp-model /path/to/ggml-large-v3-turbo.bin

# 批量任务才建议自动降级到 whisper.cpp
python video2blog.py /path/to/分享.mp4 \
  --engine auto \
  --fallback-policy auto \
  --whisper-cpp-model /path/to/ggml-large-v3-turbo.bin

# 已有字幕 / 文字稿：规整成统一 .txt / .srt / .meta.json
python video2blog.py /path/to/分享.mp4 \
  --engine external \
  --source /path/to/分享.srt

# 设好「视频输入文件根」后，可用相对路径指「单次输入文件」
export VIDEO2BLOG_INPUT_ROOT=~/Movies/inbox
python video2blog.py 分享.mp4

# 或单次命令指定根目录
python video2blog.py --input-root ~/Movies/inbox 子目录/foo.mp4

# 监听根目录本身（须先有输入根；-w 后不带路径）
python video2blog.py --input-root ~/Movies/inbox -w

# 监听根目录下某一子文件夹（路径相对根）
python video2blog.py --input-root ~/Movies/inbox -w 子目录

# 与以往等价：监听某绝对路径目录
python video2blog.py -w ~/Movies/inbox
```

说明：

- **视频输入文件根**：`VIDEO2BLOG_INPUT_ROOT` 或 `--input-root`，用于拼接 **相对路径** 的「单次输入文件」与 `-w DIR`。
- **`-w` 省略 DIR**：等价于监听该输入根本身；此时**必须**配置输入根。
- 支持扩展名：**`.mp4`、`.mov`、`.mkv`**
- 产出路径：脚本默认 **`<视频所在目录>/output/`**；**五分结构下请加 `--output-dir work/asr`**，让 ASR 产物落到中转侧而非输入侧（详见 `memory/CONFIG.md`）
- 每次成功会写三份产物：**`.txt`、`.srt`、`.meta.json`**。`meta.json` 会记录 `engine`、模型、来源置信度等，供 Step 7 审计。
- 默认 `--engine auto --fallback-policy ask`：优先 `mlx-whisper`，若当前环境无法访问 Metal 或缺依赖，会询问你：在普通 macOS Terminal 里跑、改用 `whisper.cpp`、改用外部字幕/文字稿，或退出。
- `--fallback-policy auto`：批量任务可用，MLX 不可用时自动降级到 `whisper.cpp`；如果没有配置 whisper.cpp 模型，会明确报出缺失项。
- `--fallback-policy stop`：严格模式，MLX 不可用时直接停止。
- `--run-in-terminal`：显式授权脚本打开普通 macOS Terminal 执行 MLX 转录，适合 Codex 沙箱拿不到 Metal、但本机硬件支持 Metal 的情况。
- `--engine external` 不做 ASR，只把已有 `.srt` / `.vtt` / `.txt` / `.md` 规整为统一产物，`meta.json` 会标记 `confidence=external_source` 与 `requires_review=true`。
- **`--watch`** 下会等非零大小文件 **连续几次体积不变** 再开转，减轻「文件还未拷完就开跑」的失败
- 若 `.txt` 与 `.srt` 已存在，默认跳过；要强跑：**`--force`**
- 脚本**只生成** `.srt` / `.txt`，**不写** `.md` 博文

## 4. Cursor / Agent：写博文（Step 3–8）

Agent 侧的强制约定写在 **`AGENTS.md`**（给模型读）。你给对话里的**可复制起手式**示例：

```
先读 memory/PREFERENCES.md 与 memory/CONFIG.md。
ENTRY → video
ROUTING → /lecture
SOURCE → （填 work/asr 里生成的 xxx.txt 路径）

请严格按 ARCH 文档技能链依次打开并执行：
.cursor/skills/video2blog/clean-transcript → extract-insights → structure-narrative → rewrite-blog → quality-check → format-output
端到端跑，步间不用问我。
```

若已是文字稿，把起手式改成：

```
ENTRY → transcript
ROUTING → /default
SOURCE → input/Text/某篇.md
```

精修稿可声明 **`light-clean`** 或 **`SKIP clean`**（须在自检里写明 Step 3 豁免原因），细则见架构版 §4.4。

## 5. 仓库里要改的个人文件

| 路径 | 用途 |
|---|---|
| `memory/PREFERENCES.md` | 人称、禁用套话、字数、平台格式（Obsidian 等） |
| `memory/CONFIG.md` | 本机收件路径、模型名等备忘 |
| `memory/HISTORY.md` | 近作标题/摘要，供 Step 7 看风格是否飘 |
| `knowledge/Styles/`、`Structures/`、`Prompts/` | 文风、叙事模板、硬约束片段；Step 5–6 应能指回这里 |

示例博文可放入 `knowledge/Examples/`（见该目录下 `README.md`）。

## 7. 常见问题

**Q：`video2blog.py` 报找不到 mlx_whisper？**  
A：`pip install -r requirements.txt`；若用虚拟环境，确认当前 `python` 与安装目标一致。

**Q：`mlx-whisper` 报 `No Metal device available`？**  
A：这是运行环境拿不到 Metal，不是机器不支持。优先使用 `--run-in-terminal` 或在普通 macOS Terminal 跑；也可配置 `whisper.cpp` 后选择 fallback。

**Q：`--engine auto` fallback 到 whisper.cpp 后提示缺模型？**  
A：下载或准备一个 whisper.cpp ggml 模型文件，然后传 `--whisper-cpp-model /path/to/ggml-*.bin`，或设置 `VIDEO2BLOG_WHISPER_CPP_MODEL`。

**Q：监听没反应？**  
A：默认**不递归子目录**，视频须直接落在监听文件夹根下；若在子目录里，可把监听路径改深一层或给脚本加递归（需另行开发）。

**Q：想和旧方案一样「一条命令出 .md」？**  
A：当前仓库主路径是「架构版」：**听写本地化 + Agent 编排**，更易迭代文风且不占转录 Key。若要坚持全自动 API 改稿，见《视频自动化工作流方案》里原脚本/API 叙述，可自行接回。
