# 体验架构优化（竞品分析 + IA 重构）

> 角色：资深全链路产品设计师视角。
> 目的：把当前"按引擎组织"的界面，重构成"按创作者心智组织"的体验架构。
> 写作日期：2026-05-30。
> **数据说明**：竞品部分因 web 检索工具临时不可用，基于我的知识（截止 2026-01）整理，**未做实时核对**——价格/功能可能已变，定稿前建议联网复核。IA 部分基于本仓库真实代码。

---

## 0. 一句话结论

当前产品的**功能**已经很完整，但**体验架构是「按引擎的状态机」组织的**（console / outline / review / final / artifacts 五个平级 tab），而不是按创作者的心智组织的。优化的核心动作只有一句：

> **让「你的成品、你的声音、在你的机器上、由你掌控」这四件差异化的事，在体验里随时被感知到；把引擎过程降为幕后。**

---

## 1. 竞品地形（按"它替用户干的活"分类）

> ⚠️ 知识截止 2026-01，未实时核对。

| 类别 | 代表 | 输出物 | 形态 | 与本产品的关系 |
|---|---|---|---|---|
| **一站式内容再利用** | Descript、Castmagic、Opus Clip、Riverside、vidyo.ai、Spext | 短视频切片 + 摘要 + show notes +（部分）博客草稿 | 云 SaaS / 订阅 | **最接近**，但：① 云端、要上传素材 ② 文章是"摘要/第三人称"腔，不是署名长文 ③ 订阅制 |
| **转录 + 会议纪要** | Otter、Fireflies、tl;dv、Granola、飞书妙记、通义听悟、**MacWhisper** | 逐字稿 / 摘要 / 待办 | 多为云；MacWhisper 本地 | 只做转录，不做"成文"。MacWhisper 是**唯一的本地可比物**，但止于转录 |
| **通用 AI 长文写作** | Jasper、Copy.ai、Notion AI、Lex、Type.ai | 通用文章 | 云 SaaS | 不锚定"你自己的口播 + 你的文风"，写出来不像你 |
| **知识捕获（消费侧）** | Glasp、Readwise、Heptabase、Obsidian | 高亮 / 摘要 / 笔记 | 本地/云混合 | **正好相反**：读者视角的消费，不是创作者视角的产出 |

### 这个产品真正占住的空位

把四个维度叠起来，几乎没有竞品同时满足：

1. **第一人称署名长文**（不是摘要、不是切片、不是读后感）
2. **基于你自己的口播**（不是凭空生成）
3. **你的文风**（可编辑的 STYLE_GUIDE + 指纹）
4. **本地优先 + 自带 Key**（素材不出机器，不卖你 token）

> Castmagic/Descript 能出博客草稿，但云 + 通用腔 + 订阅；MacWhisper 本地但只转录。**"本地 + 署名长文 + 你的声音 + 你掌控"这个组合是空的**——这就是该产品对「创作者 / 知识用户 / Dev 同行」三类人的全部说服力来源。

### 战略含义（直接决定 IA 的北极星）

体验里要**不断复述这四个差异点**，而不是把它们埋在功能里：
- "听起来像你"（文风/声音是主角，指纹可见）
- "留在你机器上"（本地是可见的信任信号）
- "是能发的文章，不是摘要"（成品前置、第一人称）
- "你说了算"（检查点、可编辑合同）——这正是 Dev/创作者买账的点

## 2. 当前体验架构审计

### 2.1 现状地图（来自真实代码）

```
Tauri 壳
├─ 侧栏（可收起 ⌘\）：recents（实时 job + 历史归档）+ 用户/连接页脚
├─ 首页（无选中 job）= Claude home 形态
│   问候「接下来，写点什么？」+ 创作概览卡（成品数/活跃天数/热力图/平均质检/常用路由）
│   + 启动器 composer「开始一篇改写…」→ 展开 CreateForm
├─ CreateForm（新建任务）：源 + 模式(full/quick) + 路由(5选) + 策略(single/sectioned)
│   + 演讲人(可自动识别) + 配置档 + max_retries + force + pause_on_outline  ← 约 8 个决策
├─ JobWorkspace（选中 job）= 5 个平级 tab：
│   运行日志(StepProgress+叙事) · 骨架审批 · 草稿与质检 · 成品 · 过程产物
└─ 独立设置窗口（⌘,）：模型配置档 CRUD + 写作知识库编辑器
```

### 2.2 七个体验问题（按影响排序）

| # | 问题 | 影响的 ICP | 本质 |
|---|---|---|---|
| P1 | **成品是产品，但旅程把机器摆在 C 位**：console/StepProgress 占主舞台，真正有价值的「成品」只是 5 个 tab 之一 | 全部 | IA 把"过程"当主角、"结果"当配角，价值感被稀释 |
| P2 | **新手第一次出稿的悬崖**：先配 Key（多配置档对新人是噪音）→ 看懂 full/quick + 5 路由 → 等转录+6 步。没有"5 分钟见效"路径 | 知识用户、创作者 | 首run 与稳态首页是同一个，没有引导态 |
| P3 | **建任务认知过载**：一个任务 8 个决策，引擎旋钮被当成平级选项暴露 | 知识用户 | 缺渐进式披露（智能默认 + 高级折叠） |
| P4 | **路由是黑话**：/lecture /dialogue /screencast /meeting /default，用户不知道为何要在意 | 全部 | 用工程概念命名，没翻成人话场景 |
| P5 | **差异化（你的声音）被埋进设置**：可编辑 STYLE_GUIDE/指纹 = 让输出"像你"的核心，却藏在设置窗的"知识库"里 | 创作者、Dev | 把核心卖点当成配置项 |
| P6 | **没有"作品集"这个一等场所**：历史成品散在侧栏 recents + 首页概览卡，没有专门浏览/重读/重导出/对比的地方 | 创作者 | 缺创作者的留存与成就感主面 |
| P7 | **质量学习闭环不可见**：Step 7 有分，但没采集"接受/小改/重写"的真实信号，工具不会"越用越像你" | 全部 | 缺反馈回路 |

## 3. 优化后的体验架构

### 3.1 顶层重构：从「5 个引擎 tab」到「3 个创作者场所 + 1 个声音」

产品的循环是 **素材 → 改写 → 成品 → 沉淀**。IA 应该镜像这个循环，而不是镜像状态机。

```
┌────────────────────────────────────────────────────────────┐
│  开始 Start     工作台 Workshop     作品集 Library          │
│  （起一篇）      （正在打磨的这篇）   （你的全部产出）        │
│                                                              │
│              你的声音 Voice（文风/偏好/指纹，一等公民）      │
│              设置 Settings（模型/Key，纯配置）              │
└────────────────────────────────────────────────────────────┘
```

- **开始 Start**：一个最显眼的动作——丢视频 / 贴文字稿。其余全部智能默认，"调整"折叠在后。首run 走样片引导。
- **工作台 Workshop**：当前这篇。**成品/草稿前置为主角**，过程（console）降为可收起的"幕后"泳道；人工节点（骨架/草稿审批）是醒目的"轮到你了"时刻。
- **作品集 Library**：全部成品的可浏览合集（创作者的作品身体），重读/复制/导出/Finder，质检一眼可见，活动热力图归位到这里。
- **你的声音 Voice**（从设置里提拔出来）：可编辑文风/偏好/范文 + 指纹可视化。"让它像我"作为一等概念。模型/Key 留在朴素的设置里。

> 注意：这不是要做 4 个重页面。可以仍是单窗 + 侧栏，但**导航的心智单位从"任务的 5 个 tab"换成"创作者的 3 个场所"**。Workshop 内部再分过程/成品。

### 3.2 P1 修复 · 成品前置（Artifact-forward）

- job 跑完后，**默认落在成品阅读视图**（已有 FinalView 的 ~72ch 居中阅读列），不是 console。
- 过程（StepProgress + 叙事日志）收成顶部一条**可展开的"幕后"带**——想看就点开，不想看不挡路。
- 草稿/骨架审批时，成品区显示"轮到你了"的明确召唤，而不是混在 tab 里。

### 3.3 P2/P3/P4 修复 · 建任务渐进式披露 + 人话

把 8 个决策压成一条主路径：

```
丢素材  →  [自动] 路由建议 · 演讲人识别 · 模式按时长推断  →  「开始」
                                                          └ 「调整」展开高级
```

- **路由翻成场景人话**（ID 不变，只改文案）：
  - 我在讲课/分享 (`/lecture`)｜我是被采访的嘉宾 (`/dialogue`)｜我在录屏讲解 (`/screencast`)｜我在主持会议 (`/meeting`)｜不确定，AI 判断 (`/default`)
- **智能默认**：路由按文件名+前 200 字建议（合同里本就有此约定）；演讲人自动识别（已有 `/api/detect-speaker`）；模式按时长/字数推断；策略 single 默认，长稿自动提示 sectioned。
- `max_retries / force / pause_on_outline / 配置档` 收进"调整"折叠区——日常用户永远不用碰。

### 3.4 P5 修复 · 把"你的声音"提拔成一等公民

- 顶层导航加 **Voice**：文风指南、写作偏好、参考范文的**表单化编辑**（不是裸 Markdown），+ **指纹可视化**（`fingerprints.jsonl` 已有数据——做一个"你的文风画像"小图：句长分布/高频词/段落密度）。
- 建任务时可一眼看到"将用你的文风（基于 N 篇）"——把差异化前置到决策点。

### 3.5 P6 修复 · 作品集（Library）

- 独立场所：卡片墙/列表，按时间/路由/质检分。
- 每篇：重读（已有阅读器）、复制全文、导出、Finder、质检分、"重跑/再改一版"。
- 首页概览卡的统计 + 热力图归位到这里（首页只保留"开始")。
- 这是创作者的**留存面**——看见自己攒下的作品，是继续用的最大动力。

### 3.6 P7 修复 · 轻量学习闭环

- 成品视图加一个**一键处置**：`👍 直接用了` / `✍️ 改了改` / `🔁 重写了`。
- 信号沉到本地（HISTORY 或新文件），未来用于：① 校准 Step 7 阈值 ② 给创作者一条"工具越来越懂你"的可见曲线。
- 零负担、不打断、纯本地——符合产品气质。

## 4. 落地优先级（适配 5–10h/周 · 工程驱动 · 单人）

按 **「用户感知收益 ÷ 工程成本」** 排，每项都能独立增量交付：

| 序 | 动作 | 成本 | 收益 | 备注 |
|---|---|---|---|---|
| 1 | **路由 → 场景人话**（P4） | 极低（改文案） | 中 | CreateForm 文案层，半小时 |
| 2 | **成品前置**（P1）：job 完成默认落成品阅读视图，过程收为可展开带 | 低 | 高 | 调 JobWorkspace 默认 tab + 折叠 console |
| 3 | **建任务渐进式披露**（P3）：主路径 + "调整"折叠高级旋钮 | 中 | 高 | CreateForm 重排，智能默认已有后端支持 |
| 4 | **作品集 Library**（P6）：把首页概览/历史升级成可浏览作品墙 | 中 | 高 | 复用 historicalJobs + 阅读器 |
| 5 | **Voice 提拔 + 指纹可视化**（P5） | 中-高 | 中-高 | 指纹数据已有；表单化编辑是新活 |
| 6 | **处置反馈闭环**（P7） | 低 | 中（长期高） | 成品视图加 3 个按钮 + 落盘 |
| 7 | **首run 样片引导**（P2） | 中 | 中 | 需内置样例 + 引导态，留后 |

> 建议先做 **1 + 2 + 3**：一周内就能让"建任务更简单、成品更像主角"两件事落地，是性价比最高的体验跃迁，且都不动引擎、风险低。

## 5. 给用户的待确认问题

1. 顶层 IA 从「任务的 5 tab」换成「3 场所 + Voice」，方向认同吗？还是更想保留当前结构、只做局部优化（1+2+3）？
2. "作品集 Library" 你觉得是刚需，还是当前侧栏 recents 已经够？（取决于你攒成品的节奏）
3. "你的声音/指纹可视化" 对你（创作者本人）有吸引力吗？还是更像 Danger 区不想暴露？
4. 要不要等 web 检索恢复后，我把竞品部分做一次实时复核（价格/新功能可能变了）？

## 6. 一句话收尾

功能不缺，缺的是**让差异化在体验里被持续感知**。最小动作：建任务更傻瓜（人话+默认+折叠）、成品当主角（前置阅读）、声音可见（指纹+表单）。这三件不动引擎、单人可做、一两周见效——是当前阶段性价比最高的体验架构升级。
