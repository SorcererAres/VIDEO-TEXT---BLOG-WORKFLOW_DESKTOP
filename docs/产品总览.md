# Video2Blog 产品总览

> 角色：PM 视角的项目说明书。
> 目的：在和"老板/合伙人/未来同事"对齐产品方向之前，先有一份"这是什么、为谁做、现在到了哪一步、边界在哪儿"的共同地基。
> 写作日期：2026-05-30。仓库当前状态见 `README.md` 与 `docs/桌面端方案.md`。

---

## 1. 一句话定位

**把口播视频、访谈、讲座、录屏讲解或现成文字稿，自动改写成"演讲人本人署名的可发布博客文章"的桌面端工作流应用。**

注意三个限定词：

- **演讲人第一人称**——这不是"AI 总结视频"，是"AI 替演讲人把口播写成他的博客"。视角不许漂到观众或编者。
- **可发布**——不是 demo 级的 markdown，是落到 `output/Posts/2026/<日期>-<中文短标题>.md`、能直接进 Obsidian / 个人博客的成品。
- **桌面端工作流**——本地跑，本地存，本地用钥匙串管 API Key，原料永不上传到第三方服务。

## 2. 为谁做（用户画像）

- **第一画像：内容生产者本人**——做播客 / 视频课 / 公开演讲 / 内部分享的人，想把已经讲过的口播沉淀为署名博客。
- **第二画像：知识工作者**——PM / 独立开发者 / AI 工作流爱好者，听了 1–2 小时的访谈讲座，想沉淀成自己消化版的笔记或文章（这是当前 `memory/HISTORY.md` 里 90% 的用法）。
- **第三画像（潜在）**：自媒体内容团队，把素人 KOL 的访谈批量做成博客转 SEO 流量。

目前真实跑过的成品 **48 篇**（见 `output/Posts/2026/`），覆盖访谈、讲座、播客、录屏讲解四种素材，验证了画像可达成。

## 3. 解决什么问题（Job to be done）

> "我有 1 小时的视频/讲座，想要 1 篇高质量的、像我自己写的、能直接发的博客。"

竞品和替代品里都有一段断层：

| 方案 | 出来的东西 | 断层 |
|---|---|---|
| 直接给 Whisper / 飞书 | 逐字稿 | 不能直接发布，散乱无观点 |
| 通用 LLM「帮我把这段总结成文章」 | 第三人称读后感 | 视角错了：是"AI 看完视频写的总结"，不是"我（演讲人）写的博客" |
| 人工编辑代写 | 高质量博客 | 慢、贵、风格不稳定 |
| Notion AI / Glasp | 摘要 + 高亮 | 仍是消费视角，不是创作视角 |

Video2Blog 的产品差异化锚点是 **"视角忠实度"**：

- 系统层面把"演讲人第一人称"写进 `STYLE_GUIDE.md` 和 `memory/PREFERENCES.md`；
- Step 7 质检里专门有一个 **视角忠实度** 维度（10 分制），低于 5 分直接判 REVIEW，不能发；
- 引擎落盘前自动跑 `VIEWER_RE` 正则扫一遍，命中"我看完""这场分享让我"等观看者套话直接拦下。

这是工程能保证的、其他工具没在做的事。

## 4. 现在能干什么（v0.1 实际落地）

### 4.1 端到端能力

```
视频 / 文字稿
  │
  ├─ Step 1–2  ffmpeg + MLX-Whisper / whisper.cpp 本地转录
  ├─ Step 3    清洗：去口水、断句、按节切片
  ├─ Step 4    提炼：观点 / 金句 / 案例 / 数据 / 疑点
  ├─ Step 5    结构：标题候选 + 大纲骨架  →  人工审批节点
  ├─ Step 6    改写：演讲人第一人称成文（支持长稿按节滚动）
  ├─ Step 7    质检：6 维 x 10 分 = 60 分，PASS / REVIEW
  └─ Step 8    落盘：PASS → 正文；REVIEW 经用户接受 → DRAFT-
```

两种模式：

- `full`：Step 3→8 全跑，长视频和正式定稿。
- `quick`：Step 6→8，已有清晰文字稿或轻量改写。

路由对"主体是谁"的判定（影响人称）：`/lecture` `/dialogue` `/screencast` `/meeting` `/default`。

### 4.2 桌面端外壳

Tauri 壳 + 系统 WKWebView（不带 Chromium），原生窗口 / 交通灯 / 跟随系统外观（浅 / 深 / 自动）/ 记住窗口尺寸 / 原生菜单 + Cmd+, / Cmd+N。

设计语言已从"系统 HIG 中性"主动转向"Claude 桌面端品牌气质"：暖象牙 / 暖炭 / 黏土珊瑚强调色，纸感非玻璃感，邀请式空态 + Claude home 形态首页（问候 + 创作概览卡 + 博文活动热力图 + 底部 composer）。

### 4.3 配置与多模型

- **LLM 多配置档管理器**：可存多个服务（DeepSeek / OpenAI / 自定义 OpenAI 兼容），各自独立 Key/模型/参数，可启用·停用，单一默认 ★。
- **Key 不明文落盘**：每档一条进 macOS 钥匙串（account=`profile:<id>`），任务对象 / SSE 日志 / 错误信息全程脱敏 `***`。
- **环境变量优先级最高**：headless / CI 走 `VIDEO2BLOG_API_KEY`。

### 4.4 知识库可视化编辑

在 app 里直接编辑驱动写作的"唯一真相"文件：`STYLE_GUIDE.md` / `PREFERENCES.md` / 路由 SKILL.md / 步骤 SKILL.md 等。改后**合同指纹自动失效旧缓存**，下一次 job 跑新合同。

### 4.5 稳定性骨架

- **检查点架构**：每步「读盘 → 处理 → 原子落盘 → 更新 state.json」，断电、关 app、API 抖动都能从最近完成的步骤续跑，绝不从头再来、绝不重复烧钱。
- **幂等键 = (输入 hash + step + 模型 + 合同版本)**，键没变直接复用缓存。
- **合同指纹**写进文章 frontmatter，可审计"哪个模型 + 哪版合同产出了这篇"。
- **自我修正闭环**：Step 6 ⇄ Step 7 至多 N 轮，择优而非取末轮。
- **故障注入测试**：API 429/500/超时、子进程中断、磁盘写一半，全部恢复或干净报错。

## 5. 边界与不做

- **不做内容消费工具**——不是"我看完视频，给我个摘要让我记笔记"。
- **不做多人协作**——单机本地工具，没有团队空间 / 评论 / 审批流（除了"自己批自己"的大纲审批）。
- **不做云端服务**——原料、产物、配置全在本机；不上传素材、不卖 token、不替你保管 Key。
- **不做发布渠道**——只产 Markdown，不直接发 Substack / WeChat / Notion；用户自取自发。
- **暂不做 Windows / Linux**——MLX 转录强依赖 Apple Silicon Metal，Phase 3 分发版默认 bundle whisper.cpp 之前，先稳住 macOS。

## 6. 现在卡在哪里（v0.1 → 真·分发版的差距）

1. **后端二进制化第一公里已通**（`make backend-bin`，PyInstaller onedir, ~24M），但 **Tauri sidecar 集成 + 签名公证** 还没做完。`.app` 目前不能开箱即用，用户得手动起 Python 后端 8765。
2. **MLX 转录的打包难题**没解：`.metallib` 资源收集是个坑，本轮主动跳过；分发版的现实选项是默认 bundle whisper.cpp，MLX 留给高级用户。
3. **质量基线没沉淀回归集**：48 篇成品没全部进 golden-set，prompt / 合同改动后只能靠新跑一篇手动对比，规模再大就压不住。
4. **冷启动到第一次出稿的门槛仍偏高**：第一次进 app 要 ① 配 API Key ② 看明白 full/quick/routing 三个选择 ③ 等转录 + 6 步 LLM。新用户对"为什么要选路由"没有直觉。
5. **没有商业模式探索**——目前是"作者自用工具 + 开源"，没决定要不要做付费、做谁的付费、做哪一段的付费。

## 7. 关键指标（建议）

如果要正式做这个产品，下面这些是值得追的：

| 维度 | 指标 | 现在水位 |
|---|---|---|
| 成品质量 | Step 7 总分 ≥ 42 / 60 且 视角忠实度 > 5 的 PASS 率 | 未系统统计，需要从 `output/Reviews/` 抽样 |
| 成本 | 单篇平均 token 与￥成本 | state.json 里有，未聚合到 dashboard |
| 时间 | 视频时长 → 成品落盘的 wall-clock | 未系统统计 |
| 留存 | 用户 7 / 30 日内是否再来跑一次 | 单机版没埋点，暂无 |
| 主动权 | 大纲审批节点的接受率（一次过 vs 改后通过） | 未统计 |

第一阶段不用全埋，先把 PASS 率、单篇成本、wall-clock 这三件事在 `FinalView` 里清楚地呈现，让用户自己看到价值。
