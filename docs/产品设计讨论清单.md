# 产品设计讨论清单

> 这是和你 1-on-1 对齐方向的"题目册"。每条都给到「我现在的判断」+「为什么这么判断」+「想听你怎么说」，让讨论尽量收敛而不是发散。
> 写作日期：2026-05-30。在做任何决定前我不会自动改代码。

## A. 定位与边界（最先要拍）

### A1. 这是"作者自用工具"还是"要给别人用的产品"？

- **我的判断**：现状是 **作者自用的高完成度工具**——48 篇成品全是你一个人在跑（HISTORY 看下来视角主体大量是"我"和"白墨西"）。如果只服务自己，那 Phase 3 的签名公证、whisper.cpp bundle、自动更新可以全部砍掉。
- **为什么**：分发版的成本和"自用 + 给少数朋友"的收益不对等；当前形态已经够你一周稳定出 5+ 篇。
- **想听你说**：你心里更想往哪个方向走？「作者工具」「小众产品（≤1000 个目标用户）」「严肃产品（万级 + 商业化）」分别意味着完全不同的接下来 3 个月。

### A2. 如果做产品，谁是 ICP（Ideal Customer Profile）？

候选画像（按可触达性 × 付费意愿排序）：

1. **B 端：B2B 内容团队 / 知识付费机构**——把员工讲课、内训、客户访谈批量做成博客 / 公众号。
2. **C 端：独立创作者 / 个人 IP**——播客、视频课、Newsletter 作者。
3. **C 端：知识管理重度用户**——把听到的访谈/讲座沉淀进自己的知识库（其实你现在就是）。

- **我的判断**：3 最像你自己（dogfooding 最稳），1 付费力最强，2 最有传播效应。
- **想听你说**：你做这个产品是想"让自己更爽"，还是"做生意"，还是"做小而美的开源工具积累影响力"？三种都是有效答案，但路径完全不同。

### A3. 一定不做什么？

我把"不做"写进了 `docs/产品总览.md §5`，希望你校对：
- 不做内容消费工具（摘要、笔记、高亮）。
- 不做多人协作。
- 不做云端服务（数据 / Key 都不出本机）。
- 不做发布渠道（只产 Markdown）。
- 暂不做 Windows / Linux。

**有没有要加进去的"不做"？或者你其实想做其中某个？**

## B. 用户体验的几个真痛点（我观察到的）

### B1. 新用户的第一次出稿门槛仍偏高

第一次进 app 要：① 配 API Key（多档管理对新人是噪音）② 看明白 `full` / `quick` 与 5 个 routing 的含义 ③ 等 1 小时视频转录 + 6 步 LLM。

- **我的判断**：可以做一个 **"5 分钟样片体验"**——内置一段 5 分钟的范例音频和一份样例文字稿，新用户不用配 Key 也能看到一篇完整成品（用本地 mock 或预生成结果）；之后再引导配 Key 跑真的。
- **想听你说**：你觉得"上手摩擦"是问题吗？还是"我们的用户本来就是会配 Key 的人，不用降低门槛"？

### B2. routing 这个概念用户能不能秒懂？

`/lecture` `/dialogue` `/screencast` `/meeting` `/default` 是 5 个对"谁是主体"的判定。但对新用户来说，他不知道为什么要在意这个。

- **我的判断**：可以把 routing 选择改成 **场景化的人话**——"我是讲课/分享的人"（lecture）/"我是被采访的嘉宾"（dialogue）/"我在录屏讲解"（screencast）/"我在主持会议"（meeting）/"不确定，AI 帮我判断"（default）；底层 ID 不变，只是 UI 文案变。
- **想听你说**：要不要做？还是保留极客感？

### B3. 6 步串行 + 一次 1 小时，进度感很弱

视频任务最长能跑 1+ 小时。`StepProgress` 已经有视觉化进度条，但用户没法预判"还要多久"。

- **我的判断**：把 `.state.json` 里 token / 累计耗时聚合，在 `FinalView` 之外再加一个"运行中"仪表盘——剩余预估时间 / 当前累计成本 / 当前步骤详情。
- **想听你说**：必要吗？还是这是过度优化？

### B4. 大纲审批节点是否对的人？

`WAITING_USER_OUTLINE` 是 full 模式默认开的，让用户改大纲再继续。

- **我的判断**：对你（作者本人）有用；对纯小白可能是 **决策疲劳**——他不知道大纲该不该改。可以加 **"AI 给出 3 个大纲候选，你只挑一个"** 的形态，把"创作决策"压缩成"选项决策"。
- **想听你说**：你自己用大纲审批多频繁？接受率多高？

## C. 质量这件事（核心壁垒）

### C1. 现在的质检到底准不准？

48 篇成品都跑过 Step 7 的 6 维评分。但是：
- 评分是 LLM 自评的（让评估模型当裁判，存在偏置）；
- 我没看到把"用户实际是否接受 / 是否手工大改"这个 ground truth 喂回来做对齐。

- **我的判断**：可以在 `FinalView` 加一个 **轻量"我接受 / 我大改了 / 我重写了"** 的反馈按钮，把这个信号长期沉到 `memory/HISTORY.md` 或新文件里，未来用来校准 Step 7 阈值。
- **想听你说**：愿不愿意给自己加这个动作？

### C2. 自修正的轮数 N 该不该开放给用户？

默认 N=1。但有时候第一稿差距很大，重试一轮也不够；有时候第一稿就够好，重试反而越改越烂。

- **我的判断**：保持默认 N=1，但开放 **"质量优先" / "速度优先"** 两个预设（背后调 N 和 temperature）。比超参数面板更人话。
- **想听你说**：值得做吗？

### C3. golden-set 怎么建？

现在 5 个 fixture 是 mock LLM 的，验证"确定性环节"。但真实质量回归没有 baseline。

- **我的判断**：从 48 篇成品里挑 **8–10 篇有代表性的**（覆盖 lecture / dialogue / screencast / 长 / 短 / 中英混排），冻结为 golden-set；之后每改一次 prompt 都跑这 8 篇 + 真实 LLM 对比新旧分。
- **代价**：每次回归一次大约 ¥X（取决于模型），但能保住质量基线。
- **想听你说**：值不值这个钱？谁来批准每次回归的预算？

## D. 商业化（可选）

### D1. 如果要做商业化，最早能卖什么？

候选：

1. **预编译 + 签名 .dmg 收一次性 fee**（开源 + 付费分发，类似 Sublime Text）。
2. **托管版**——用户上传视频 / 文字稿，云端跑，按 token 计费 + 加成。**但这违反"数据不出本机"的初心，慎选**。
3. **企业版**——B2B SaaS，做团队版 + SSO + 私有部署。需要的工作量是 10×。
4. **不卖产品，卖服务**——把你跑过 48 篇的经验做成"内容生产顾问"，工具是 lead magnet。

- **我的判断**：现阶段什么都不卖，先把 ICP 和 PMF 想清楚；如果一定要选，**1 最干净**，因为不动初心。
- **想听你说**：你对商业化有什么"必须"或"绝不"的态度？

### D2. 开源策略

现在是私仓还是公开？我没翻 `.git/config` 确认。

- **想听你说**：要不要开源？什么时机？开源是为了拿用户、拿 contributor、还是拿名声？

## E. 下一个 30 天投什么（最实操）

按 "用户最痛 × 工程最小" 排，我的建议优先级：

1. **A 选择"自用 / 小众 / 严肃"定位**——这一条不拍，后面全是猜。
2. **如果选小众或严肃**：做 Tauri sidecar 集成 + 签名公证（解封"开箱即用"）。
3. **加 routing 的人话文案 + 5 分钟样片体验**（降低首启门槛）。
4. **在 FinalView 暴露单篇成本 + wall-clock**（让用户看见价值 & 帮你之后讲产品）。
5. **建 8–10 篇 golden-set + 加"我接受 / 大改 / 重写"反馈**（沉淀质量基线）。
6. **写一份给"非你本人"看的 README**（很多决策都被"你已经懂"挡住了）。

如果选自用，那只做 4 + 5 就够了，其他全部砍。

## F. 我没把握的事，想先问你

1. 你做这个工具的**原始动机**是什么？"我自己要用" / "我有 X 个朋友想用" / "我想做产品"？
2. 你愿意每周投多少时间在这个项目上？（决定上面 E 列表能砍多狠）
3. 你目前最不满意的地方是什么？（让我别拍脑袋猜你的痛点）
4. "演讲人第一人称"这个差异化锚点，是你**深思熟虑**的产品判断，还是**只是当前工作流的副产品**？这件事影响很多。
5. 桌面端这个形态（vs 网页版 / vs CLI），是路径依赖，还是产品选择？
