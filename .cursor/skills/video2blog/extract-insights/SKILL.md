# extract-insights

## 何时使用

- Step 3 已完成（或豁免已记录）。
- `ROUTING` 决定取舍权重（讲演偏论点，对谈偏交锋，录屏偏步骤——见架构版 §4.3）。

## 执行前必读

- `Knowledge/Structures/` 中对应路由提纲（至少在 Step 5 展开前你心里要有目录）。

## 输入

- Step 3 的「清洗稿」全文

## 输出格式（Markdown，勿用 JSON）

```markdown
## 核心观点（3–5）

1. …
## 原话金句（3–7）

- 「…」
## 案例与故事

- …
## 数据与事实

- …
## 待确认项

- [?] …
```

## 步骤

1. 先列出高置信观点，再给支撑案例/数据挂钩。
2. 金句保持可引用长度；过长则截取 + 省略号。
3. 每条待确认项指向清洗稿位置或复述关键词。

## 反例

- 用模型常识补案例（无文本依据）。
