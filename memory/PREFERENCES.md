## 语言与受众

- 正文语言：**简体中文**
- 受众：PM、独立开发者、AI Agent / 工作流爱好者；默认读者懂基本技术名词

## 人称与语气

- 叙述人称：**演讲人第一人称「我」**——博文是把视频里说话人的口播改写成他署名的博客，不是观众读后感
- **"演讲人"按路由定义**（详见 `knowledge/ROUTER.md`「演讲人主体」节）：
  - `/lecture` → 演讲者 ｜ `/dialogue` → 嘉宾（输出主要观点的一方，不是主持人）
  - `/screencast` → 录屏者 ｜ `/meeting` → 主持或主决策者
  - 用户可在指令中写 `SPEAKER → 某某` 覆盖
- **不许混入观看者视角**：禁止出现「我看完这场访谈」「这场分享让我学到」「我抄走的几句」「我作为读者觉得」「跨篇对照」「编者按」「我的补充观察」等外部评论性语句
- **不许反驳演讲人自己**：博文是演讲人署名作品，反驳/批判内容必须重写为"我（演讲人）后来又想到的反例"或干脆不写
- 语气：介于口语与书面之间；可有观点与节奏感，但不口水

## 版式与平台

- 输出格式：**Obsidian Markdown**（可选用 `[[wiki link]]`，若不用请删本行）
- 小标题层级：正文用 `##`，子节用 `###`

## 长度

- 目标字数：**1500–3000 字**（超长逐字稿可分段成文，须在 `format-output` 说明分段策略）

## 禁用套话（须删除）

- 「大家好」「感谢收看」「以上就是」「今天我们来聊」「点赞收藏」类口播首尾套话与求互动话术（可按需增删）

## 专有名词

- 英文术语：**保留英文原词**，必要时中文括注一次性解释
- 中英混排：**中文与英文/数字之间加空格**（如：使用 Whisper 模型）
