# 项目结构说明

> 本文件是**目录速查**。运行契约见 [`knowledge/工作流契约.md`](knowledge/工作流契约.md)，Agent 强约束见 [`AGENTS.md`](AGENTS.md)，上手命令见 [`使用说明.md`](使用说明.md)，设计背景见 [`Archive/视频博文工作流-架构版.md`](Archive/视频博文工作流-架构版.md)（已归档）。

本仓库把「视频/文字稿 → 可发布博文」拆成一条流水线，目录按职责分成**五侧**，输入 / 中转 / 输出三侧严格不互串。

## 五分结构总览

| 目录           | 角色                                               | 谁读写                   |
| -------------- | -------------------------------------------------- | ------------------------ |
| `input/`     | **输入侧**：用户喂进来的原料                 | 用户放料；脚本读         |
| `knowledge/` | **写作配方**：静态知识库，不参与运行只被引用 | 用户维护；Agent 读       |
| `memory/`    | **读侧记忆**：用户偏好 / 配置 / 历史         | 用户维护；Agent 每次先读 |
| `work/`      | **流水线中转**：ASR 产物与规整稿             | 脚本写；Agent 读         |
| `output/`    | **写侧成品**：博文定稿与质检报告             | Agent 写                 |

## 目录树

```
视频/文本-博文工作流/
├── video2blog.py              # ENTRY→video：本地听写（Step 1–2），不占 API
├── requirements.txt           # mlx-whisper、watchdog 等依赖
├── AGENTS.md                  # Agent 入口约定与不变量
├── 使用说明.md                 # 人类上手：安装、命令、起手式
├── 项目结构.md                 # 本文件
│
├── input/                     # ① 输入侧（喂料，输入/输出不互串）
│   ├── Video/                 #    待转写视频 .mp4/.mov/.mkv（git-ignored，仅占位入库）
│   └── Text/                  #    现成文字稿/逐字稿/纪要（ENTRY→transcript 源稿）
│
├── knowledge/                 # ② 写作配方（静态，被 Step 5–7 引用）
│   ├── ROUTER.md              #    路由→Structure/Style 唯一映射来源
│   ├── Structures/            #    叙事骨架：pyramid / scqa / debate / tutorial-flow
│   ├── Styles/                #    文风皮肤：casual-blog / deep-dive / tutorial / decision-log
│   ├── Prompts/               #    永久硬约束片段（如 zh-cn-mix）
│   └── Examples/              #    少量高质量成品，作 few-shot/对照
│
├── memory/                    # ③ 读侧记忆（Agent 每次响应前必读）
│   ├── PREFERENCES.md         #    语言、人称、字数、禁用套话、平台格式
│   ├── CONFIG.md              #    收件路径、ASR 引擎、模型名等管线配置
│   └── HISTORY.md             #    近 10 篇指纹，供 Step 7 风格一致性比对
│
├── work/                      # ④ 流水线中转（脚本写 / Agent 读）
│   ├── asr/                   #    原始 ASR 产物 .txt/.srt/.meta.json/.log
│   └── Transcripts/           #    规整后的统一文字稿
│
├── output/                    # ⑤ 写侧成品（Agent 落盘，输入/输出不互串）
│   ├── Posts/<YYYY>/          #    定稿 <YYYY-MM-DD>-<slug>.md；未过加 DRAFT- 前缀
│   ├── Reviews/               #    Step 7 六维质检报告 <slug>.review.md
│   └── README.md              #    写侧落盘契约与命名约定
│
├── .cursor/skills/video2blog/        # Cursor 版 SKILL：Step 3–8 六个子技能
└── .codex/skills/video2blog-workflow/ # Codex 版 SKILL + preflight 自检脚本
```

## 数据怎么流

```
                         knowledge/  ─┐ (Step 5–7 引用配方)
                         memory/    ─┤ (每步先读偏好/配置/历史)
                                     ▼
 input/Video/*.mp4  ──video2blog.py──►  work/asr/*.txt   ──┐
 input/Text/*.md  ───────(已有文字稿，跳过脚本)─────────────┤
                                                            ▼
                                          Step 3–8（清洗→提要→结构→改写→质检→落盘）
                                                            ▼
                                          output/Posts/  +  output/Reviews/
                                                            │
                                                            └─► memory/HISTORY.md 追加指纹
```

- **ENTRY → video**：跑 `video2blog.py` 转写。**须带 `--output-dir work/asr`**，否则脚本默认写 `input/Video/output/`（落回输入侧，破坏分离）。
- **ENTRY → transcript**：已有文字稿放 `input/Text/`，不跑脚本，Agent 从 Step 3 直接起。
- 路由（`/lecture` `/dialogue` `/screencast` `/meeting` `/default`）→ 结构+文风，唯一查 `knowledge/ROUTER.md`。

## 三条铁律

1. **输入 / 输出不互串**：`input/` 是用户喂的料，`output/` 是 Agent 写的成品，中转物一律走 `work/`。
2. **视角铁律**：博文一律以「演讲人本人」为「我」，禁观看者/编者视角与跨视频引证（详见 `memory/PREFERENCES.md`）。
3. **多宿主同源**：`.cursor` 与 `.codex` 下的 SKILL 执行同一条 `ROUTER.md` 驱动的链路，模式切换不改变约定。
